# 智能笔记提取

## 核心原则

**只记精华，不记常识。** 一篇文章的价值不在于它说了什么，在于它说了什么别人没说过的。

### 什么值得记

- **流程设计**：做一件事的完整步骤编排，尤其是步骤间的依赖和顺序逻辑
- **创新点**：某个环节的非常规做法，打破常规认知的发现
- **关键成功因素**：把事做成的核心要素，去掉就会失败的东西
- **可复用提示词/模板**：能直接拿来用的 prompt、配置模板、角色定义等
- **踩坑经验**：别人踩过的坑和解决方案，尤其是非显而易见的
- **独特洞察**：反直觉的结论、被忽视的关联、深层因果

### 什么不记

- 工具的基本介绍和官方文档已有的内容
- 众所周知的最佳实践（除非有独特视角）
- 泛泛而谈的趋势预测
- 作者的自我介绍和背景铺垫
- 重复论证同一个观点的多个段落（只取最精炼的表述）

## 工作流程

### 第一步：获取内容

**判断输入类型：**

- **直接文章内容**：用户粘贴了文章正文 → 直接进入第二步
- **URL 链接**：用户只提供了链接 → 用 `mcp__web-reader__webReader` 抓取正文，失败则用 `WebFetch` 备选
- **本地文件路径**：用户给了文件路径 → 用 Read 工具读取

### 第二步：通读理解

**必须完整阅读全文后再动笔。**

阅读时回答三个问题：
1. 这篇文章的核心主张是什么？（一句话）
2. 作者做了什么别人没做过的事？
3. 哪些内容是训练数据中不太可能覆盖的实践经验？

**如果有看不懂的地方，立即用 AskUserQuestion 询问用户，不要猜测。**

### 第三步：分类萃取

按以下维度扫描文章，每个维度独立判断是否有值得记录的内容：

| 维度 | 产出位置 | 判断标准 |
|------|---------|---------|
| 核心知识点 | `笔记/{分类}/` | 有独特发现或创新做法 |
| 可复用提示词/模板 | `笔记/prompts/` | 能直接拿来用的文本块 |

**分类规则**：参考 [references/categories.md](references/categories.md)

### 第四步：撰写笔记

**笔记格式：**

```markdown
# {主题标题}

> 来源：{作者}《{文章标题}》{日期}
> 核心主张：{一句话概括}

## {知识点1标题}

{精炼内容，保留关键数据和具体做法}

## {知识点2标题}

...
```

**写作纪律：**

- 用自己的语言重新组织，不是复制粘贴原文
- 保留具体的数据、代码、配置（这些是精华）
- 删除过渡句、铺垫段、重复论证
- 表格优于大段文字
- 有代码/配置/prompt 的，保留原文格式

### 第五步：提示词提取

扫描文章中的可复用文本块：角色定义、工作流程模板、评审标准、检查清单、配置模板。

每个提示词单独一个文件，保存到 `笔记/prompts/`，文件名：`{主题}-{用途}.md`

格式：
```markdown
# {用途描述}

> 来源：{出处}
> 用途：{什么场景下使用}

## 提示词

{原文提示词内容，保留原始格式}

## 使用说明（可选）

{效果对比、注意事项等}
```

### 第六步：去重检查与保存

1. 用 Glob 检查 `笔记/` 下是否已有同主题笔记
   - **已有**：更新到同一份文件，补充新内容
   - **没有**：创建新文件
2. 将文件写入对应目录
3. `git add` → `git commit` → `git push`

---

## 输出规范

完成后向用户报告：

```
已记录 N 份笔记：
- 笔记/{分类}/{文件名}.md — {一句话说明}
- 笔记/prompts/{文件名}.md — {用途}
```
