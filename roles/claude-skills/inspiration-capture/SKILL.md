# 灵感记录器 (Inspiration Capture)

捕捉灵感、发散思维、提升创意质量。

## 核心功能

### 1. 记录灵感

当用户表达要记录灵感时：

**步骤 1：记录基本信息**
将灵感以 JSONL 格式追加到 `data/ideas.jsonl`：

```json
{
  "id": "时间戳-hash",
  "timestamp": "2026-02-21T10:30:00+08:00",
  "title": "灵感标题（用户原话或提炼）",
  "content": "灵感详细描述",
  "category": "分类（自动推断）",
  "tags": ["标签1", "标签2"],
  "status": "raw",
  "questions": [],
  "related_ideas": [],
  "references": []
}
```

**步骤 2：激发讨论**

向用户提出 2-3 个深化问题：
- 这个灵感解决什么问题？
- 有什么应用场景？
- 和现有方案有什么不同？
- 有什么潜在挑战？

**步骤 3：搜索类似想法**

使用 WebSearch 搜索：
- `"{灵感关键词}" idea innovation`
- `"{灵感关键词}" 类似 产品 方案`

总结搜索结果：
- 有没有类似的产品/想法？
- 别人是怎么做的？
- 我们可以有什么不同？

**步骤 4：发散思维**

基于记录和搜索，提供：
- 3 个延伸方向
- 2 个可能的变体
- 1 个跨界组合建议

### 2. 查询灵感

当用户要查询灵感时：

**步骤 1：初步搜索**

根据用户输入，先按理解查找几个匹配项：

```
用户: "查看最近关于AI的灵感"
→ 先搜索关键字包含"AI"的记录
```

**步骤 2：展示选项**

```
找到 3 条相关灵感：

1. [2026-02-20] AI助手自动整理笔记的想法
2. [2026-02-18] 用AI生成测试用例
3. [2026-02-15] AI驱动的代码审查工具

请选择查看详情，或输入关键字进一步筛选：
- 按时间: 最近一周 / 最近一月 / 指定日期范围
- 按分类: 技术 / 产品 / 生活 / 学习
- 按状态: raw / developing / mature / archived
```

**步骤 3：展示详情**

用户选择后，展示完整信息并询问是否要：
- 继续深化这个灵感
- 合并到其他灵感
- 标记状态变更

### 3. 列出灵感

简洁列表视图：

```
灵感列表 (共 15 条)

最近 7 天:
• [02-21] 灵感标题1 #技术 #AI
• [02-20] 灵感标题2 #产品

最近 30 天:
• [02-15] 灵感标题3 #学习
...
```

## 数据格式

### 分类预设

| 分类 | 说明 |
|------|------|
| tech | 技术类：编程、工具、系统 |
| product | 产品类：功能、设计、商业模式 |
| life | 生活类：习惯、效率、体验 |
| learning | 学习类：知识、技能、方法 |
| creative | 创意类：内容、艺术、表达 |
| business | 商业类：机会、合作、策略 |

### 状态流转

```
raw → developing → mature → archived
  ↓         ↓
trashed ← ─ ─
```

- `raw`: 原始灵感，刚记录
- `developing`: 正在发展完善
- `mature`: 成熟可执行
- `archived`: 已归档（完成或放弃）
- `trashed`: 已删除

## 存储位置

```
~/.claude/skills/inspiration-capture/data/ideas.jsonl
```

每行一条 JSON 记录，追加写入。

## 操作指令

| 用户说 | 执行 |
|--------|------|
| "记录灵感..." | 触发记录流程 |
| "查询灵感" | 进入查询模式 |
| "列出灵感" | 显示列表 |
| "查看 [id/标题]" | 显示详情 |
| "更新 [id]" | 修改灵感 |
| "删除 [id]" | 移到 trashed |
