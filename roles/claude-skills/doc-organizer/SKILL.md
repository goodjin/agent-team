# 文档整理工具

整理笔记项目中的 Markdown 文档，执行分类、移动、合并、清洗、重命名和标记操作。

## 保护目录

以下目录**禁止修改**：
- `skills/` - 工具专用目录
- `.claude/` - 系统目录
- `.git/` - Git 目录
- `data/` - 数据文件目录（非文档）
- 用户指定的其他保护目录

## 目录规范

### 目录用途定义

| 目录 | 用途 | 典型内容 |
|------|------|----------|
| `笔记/技术/` | 技术学习笔记 | 编程语言、框架、工具教程 |
| `笔记/anthropic/` | Anthropic/Claide 相关 | Claude、API、Agent 相关笔记 |
| `笔记/工作/` | 工作相关笔记 | 会议记录、项目记录 |
| `归档/` | 历史文档 | 过时内容、旧版本、历史记录 |
| `收藏/` | 有价值的收藏 | 从网上收藏的技术文章、教程 |
| `项目/` | 项目相关文档 | 项目方案、技术调研、需求文档 |
| `docs/` | 项目说明文档 | 项目 README、说明文档 |
| `articles/` | 原创/转载文章 | 博客文章、技术分享 |

### 不应放文档的目录

| 目录 | 说明 | 处理方式 |
|------|------|----------|
| `data/` | 数据文件专用 | 移动文档到合适目录 |
| 根目录 | 仅保留 README.md | 移动其他文档到合适目录 |
| 临时目录 | 如 `tmp/`、`temp/` | 移动或删除 |

## 整理标记

已整理的文档在开头添加标记：
```markdown
<!-- organized: true -->
```

检查此标记避免重复整理。

## 工作流程

### 1. 扫描文档

```bash
find . -type f -name "*.md" | grep -v "^\./\.claude" | grep -v "^\./skills" | grep -v "^\./\.git" | grep -v "^\./data"
```

### 2. 检查目录位置

对每个文档检查当前目录是否合适：

**位置检查规则**：
1. 根目录：仅允许 `README.md`，其他文档需移动
2. `data/` 目录：不应有 `.md` 文档，需移动
3. 根据文档内容判断是否在正确分类目录

**内容分类判断**：
- 标题/内容包含 "Rust/Python/Java/Go" 等 → `笔记/技术/`
- 标题/内容包含 "Claude/Anthropic/Agent" → `笔记/anthropic/`
- 标题/内容包含 "项目/方案/调研" → `项目/`
- 标题/内容包含 "述职/报告/年度" → `归档/`
- 从网上复制的教程/文章 → `收藏/`

**移动操作**：
1. 识别需要移动的文档
2. 确定目标目录（如不存在则创建）
3. 移动文档到目标目录
4. 记录移动操作

### 3. 内容清洗

对于从网上复制的文档，执行以下清洗：

**去除内容**：
- 多余的 HTML 标签（`<div>`, `<span>` 等）
- 无意义的注释（如 `<!-- more -->`, `<!-- comments -->`）
- 复制时的格式残留（如多余的空行、特殊字符）
- 广告、推广链接
- 无关的版权声明

**格式统一**：
- 标题层级规范（一级标题仅一个）
- 列表格式统一
- 代码块添加语言标识
- 链接格式统一

### 4. 内容完善

检查并补充：
- **标题**：确保有一级标题，简洁明确
- **日期**：如需要可添加创建/更新日期
- **结构**：补充缺失的章节
- **摘要**：长文档添加简要说明

### 5. 命名规范

文件命名规则：
- **中文优先**：内容为中文时用中文命名
- **英文例外**：内容为英文时用英文命名
- **简洁明确**：文件名能表达内容含义
- **避免特殊字符**：不使用 `/\:*?"<>|` 等字符

### 6. 文档合并

合并条件：
- 内容相似度高的文档
- 同一主题的多个碎片文档
- 重复或副本文档

**合并方式**：
1. 识别内容相似的文档
2. 询问用户确认是否合并
3. 合并到较完整或命名更好的文档
4. 删除冗余副本

### 7. 添加整理标记

在文档开头添加：
```markdown
<!-- organized: true -->
```

### 8. 完整性检查

```bash
# 查找未整理的文档
find . -type f -name "*.md" | grep -v "^\./\.claude" | grep -v "^\./skills" | grep -v "^\./\.git" | grep -v "^\./data" | while read f; do
  if ! grep -q "<!-- organized: true -->" "$f"; then
    echo "$f"
  fi
done
```

**检查结果**：
- 如有未整理文档 → 标记为 **FAIL**，继续整理
- 全部已整理 → 标记为 **SUCCESS**

```
### 其他操作

[列出主要操作]

[列出需要手动处理的文档]
```

## 注意事项

1. **保护目录不改动**：skills/, .claude/, .git/, data/ 等目录不做任何修改
2. **确认后执行**：合并、删除操作先询问用户
3. **保留原始信息**：合并时保留所有有价值的内容
4. **循环检查**：FAIL 状态必须继续整理直到 SUCCESS
5. **目录位置优先**：先检查并移动放错位置的文档
