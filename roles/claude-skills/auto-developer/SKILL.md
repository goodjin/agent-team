# Auto Developer - 自动化开发编排

顶层流程编排器，协调从需求到交付的完整开发流程。

## 核心理念

### 固定的三板斧流程

```
┌─────────────────────────────────────────────────────────────┐
│                     自动化开发流程                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  第一板斧: PRD + 架构                                        │
│  ┌───────────────┐    ┌───────────────┐                    │
│  │  prd-builder  │ ─▶ │ architecture  │                    │
│  │  (行业研究)    │    │  -designer    │                    │
│  │  (竞品分析)    │    │  (架构+规约)   │                    │
│  └───────────────┘    └───────────────┘                    │
│                                                             │
│  第二板斧: 开发计划                                          │
│  ┌───────────────┐                                          │
│  │ dev-plan-     │                                          │
│  │  builder      │                                          │
│  │ (前端+后端    │                                          │
│  │  +测试计划)   │                                          │
│  └───────────────┘                                          │
│                                                             │
│  第三板斧: 执行 + 审计                                       │
│  ┌───────────────┐    ┌───────────────┐                    │
│  │ task-executor │ ─▶ │ test-auditor  │                    │
│  │ (原子化任务)   │    │ (测试审计)    │                    │
│  │ (断点续跑)    │    │ (质量门禁)    │                    │
│  └───────────────┘    └───────────────┘                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 执行模式

| 模式 | 描述 | 适用场景 |
|-----|------|---------|
| **手动模式** | 每个阶段需用户确认 | 复杂项目、首次开发 |
| **单阶段自动** | 当前阶段自动执行 | 熟悉项目、快速迭代 |
| **连续执行** | 全流程自动执行 | 标准化项目、紧急交付 |

### 唯一真相来源（SSOT）

- PRD文档是需求的唯一真相来源
- 所有下游文档严格对齐PRD
- 任何变更必须追溯到PRD

## 触发条件

用户输入包含以下内容时触发：
- "启动项目"、"执行项目"、"开发项目"
- "完整开发"、"自动交付"
- "从头开始"、"完整流程"
- 或任何需要完整开发流程的请求

## 技能协调

### 技能清单

| 技能 | 职责 | 输入 | 输出 |
|-----|------|------|------|
| prd-builder | PRD文档生成 | 用户需求 | PRD文档 |
| architecture-designer | 架构设计+规约 | PRD文档 | 架构文档+技术规约 |
| dev-plan-builder | 开发计划生成 | 架构文档 | 开发计划+任务文档 |
| task-executor | 任务执行 | 开发计划 | 代码+执行报告 |
| test-auditor | 测试审计 | 代码 | 测试报告 |

### 协调流程

```yaml
流程编排:
  1. prd-builder:
     input: 用户需求
     output: docs/v{N}/01-prd.md
     validation: PRD完整性检查

  2. architecture-designer:
     input: docs/v{N}/01-prd.md
     output: docs/v{N}/02-architecture/
     validation: 100%覆盖检查

  3. dev-plan-builder:
     input: docs/v{N}/02-architecture/
     output: docs/v{N}/03-dev-plan/
     validation: 任务大小检查

  4. task-executor:
     input: docs/v{N}/03-dev-plan/
     output: docs/v{N}/04-execution/
     validation: 测试通过检查

  5. test-auditor:
     input: 代码
     output: docs/v{N}/05-test-report/
     validation: 质量门禁检查
```

## 工作流程

### 阶段 1: 项目初始化

#### 1.1 创建项目结构

```
docs/
└── v{N}/
    ├── 01-prd.md                    # PRD文档
    ├── 02-architecture/             # 架构设计
    │   ├── 00-coverage-report.md
    │   ├── 01-overview.md
    │   ├── 02-layers.md
    │   ├── 03-*.md (模块文档)
    │   ├── 04-api-spec.md
    │   ├── 05-data-spec.md
    │   ├── 06-state-spec.md
    │   └── 07-boundary-spec.md
    ├── 03-dev-plan/                 # 开发计划
    │   ├── 00-overview.md
    │   └── *.md (计划文档)
    ├── 04-execution/                # 执行记录
    │   ├── state.json
    │   ├── logs/
    │   └── evidence/
    └── 05-test-report/              # 测试报告
        └── report.md
```

#### 1.2 确定版本号

```
检查 docs/ 目录下的 v* 文件夹:
- 当前最大版本: v3
- 新版本: v4
```

#### 1.3 确定执行模式

询问用户选择执行模式:
```
请选择执行模式:
1. 手动模式 - 每个阶段需确认（推荐新项目）
2. 单阶段自动 - 当前阶段自动执行
3. 连续执行 - 全流程自动执行（推荐标准化项目）
```

## 命令参考

### 启动项目

```bash
# 手动模式
auto-developer --mode=manual

# 单阶段自动
auto-developer --mode=stage

# 连续执行（全自动）
auto-developer --mode=continuous
```

### 从特定阶段开始

```bash
# 从架构设计开始
auto-developer --start=architecture

# 从开发计划开始
auto-developer --start=dev-plan

# 从任务执行开始
auto-developer --start=execution
```

### 恢复执行

```bash
# 从断点恢复
auto-developer --resume

# 查看当前状态
auto-developer --status
```

## 注意事项

1. **SSOT原则**: 所有决策必须追溯到PRD
2. **原子化执行**: 任务大小严格限制
3. **测试先行**: 每个任务必须测试通过
4. **断点续跑**: 支持中断恢复
5. **质量门禁**: 必须通过才能交付
