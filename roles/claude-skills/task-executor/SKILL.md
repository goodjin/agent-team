# Task Executor - 任务执行器

执行开发计划中的原子化任务，支持断点续跑、进度追踪和自动测试。

## 核心原则

### 1. 原子化执行

**任务执行限制**（严格约束）：
- **代码变更**: ≤ 200行
- **涉及文件**: ≤ 5个
- **测试用例**: ≤ 10个
- **执行时间**: ≤ 30分钟

**超限处理**：
- 如果发现任务会超出限制，立即停止并报告
- 建议将任务拆分为更小的子任务

### 2. 断点续跑

- 使用持久化状态文件记录进度
- 中断后可从上次停止位置继续
- 每个任务完成后立即保存状态

### 3. 测试先行

- 每个任务完成后立即运行相关测试
- 测试失败必须在当前任务内修复
- 不遗留技术债务

### 4. 契约门控

- 后端API变更必须更新OpenAPI规范
- 前后端接口变更必须通过契约验证
- 破坏性变更必须显式确认

## 工作流程

### 阶段 1: 初始化/恢复

#### 1.1 检查状态文件

```
检查 docs/v{N}/04-execution/state.json 是否存在:
- 存在 → 读取状态，从断点恢复
- 不存在 → 初始化新的执行状态
```

#### 1.2 读取开发计划

```
读取:
- docs/v{N}/03-dev-plan/00-overview.md
- 当前执行的开发计划文件
```

#### 1.3 确定执行模式

| 模式 | 描述 | 触发条件 |
|-----|------|---------|
| 手动执行 | 每个任务需用户确认 | 默认模式 |
| 单阶段自动 | 当前计划自动执行 | 用户指定 `--auto` |
| 连续执行 | 全部计划自动执行 | 用户指定 `--full-auto` |

### 阶段 2: 任务执行循环

```
┌─────────────────────────────────────────────────────────┐
│                    任务执行循环                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  while (有待执行任务) {                                  │
│      1. 读取下一个任务                                   │
│      2. 检查任务依赖                                     │
│      3. 执行前置检查                                     │
│      4. 执行任务                                        │
│      5. 运行测试                                        │
│      6. 契约门控检查                                    │
│      7. 更新状态                                        │
│      8. 记录证据                                        │
│  }                                                      │
│                                                         │
│  if (所有任务完成) {                                     │
│      生成执行报告                                        │
│  }                                                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

```markdown
从开发计划中读取任务详情:
- 任务编号

- 输入/输出
- 实现要求
- 验收标准
- 测试要求
- 依赖任务
```

#### 2.2 检查依赖

```markdown
## 依赖检查

| 依赖任务 | 状态 | 检查结果 |
|---------|------|---------|
| T-01 | ✅ 已完成 | 通过 |
| T-02 | ✅ 已完成 | 通过 |

**依赖检查通过** → 继续执行
**依赖检查失败** → 停止并报告
```

#### 2.3 前置检查

```markdown
## 前置检查清单

- [ ] 依赖任务已完成
- [ ] 所需文件/目录存在
- [ ] 测试环境可用
- [ ] 无阻塞问题
- [ ] 项目特定验证规范已读取（如有）
```

**项目特定验证规范**:

如果开发计划中包含项目特定验证规范（如平台特定行为检查清单），必须在执行前读取并应用。

例如 SwiftUI macOS 项目的验证规范可能包括:
- HSplitView 子视图必须有明确的 width 约束
- 自定义工具栏需要考虑系统工具栏高度
- 点击区域必须使用 .contentShape(Rectangle()) 确保

#### 2.4 执行任务

**执行步骤**:

1. **读取任务文档**
   - 完整读取任务定义
   - 理解实现要求
   - 确认验收标准

2. **创建执行前快照**
   ```
   记录:
   - 将要修改的文件列表
   - 当前文件内容
   - 测试状态
   ```

3. **实现任务**
   - 按照任务定义编写代码
   - 严格限制代码变更量
   - 遵循代码规范

4. **创建执行后快照**
   ```
   记录:
   - 修改后的文件内容
   - 新增的文件
   - 删除的文件
   ```

#### 2.5 运行测试与验证

**验证类型分层**:

| 验证类型 | 适用场景 | 验证方法 |
|---------|---------|---------|
| 静态验证 | 所有任务 | 编译通过、类型检查、lint |
| 逻辑验证 | 业务逻辑任务 | 单元测试通过 |
| 视觉验证 | UI布局任务 | 截图对比设计规范 |
| 交互验证 | 用户交互任务 | 手动点击测试 |

**测试执行顺序**:

```
1. 静态验证
   - 编译检查：无编译错误
   - 类型检查：无类型不匹配
   - 代码规范：符合项目规范

2. 逻辑验证（单元测试）
   - 只运行与当前任务相关的测试
   - 记录测试结果

3. 视觉/交互验证（如任务需要）
   - 检查验收标准中的视觉验证项
   - 检查验收标准中的交互验证项

4. 分析验证结果
   - 全部通过 → 继续
   - 有失败 → 修复并重试（最多3次）
   - 3次仍失败 → 标记任务失败，报告问题
```

**验收标准检查**:

每个任务的验收标准必须使用**三级验证格式**:
- `[ ] 数据展示验证`: 静态渲染正确、空状态处理
- `[ ] 布局验证`（容器类/组件类）: 尺寸符合设计规范、不挤占核心内容区
- `[ ] 交互验证`（交互类）: 事件触发验证、状态反馈验证、数据流转验证

执行任务时必须逐项检查验收标准，未通过的项必须在当前任务内修复。

**测试结果记录**:

```markdown
## 测试结果

### 单元测试
- 测试文件: __tests__/service.test.ts
- 用例数: 8
- 通过: 8
- 失败: 0
- 覆盖率: 85%

✅ 测试通过
```

#### 2.6 契约门控检查

**适用场景**: 涉及API变更的任务

**检查步骤**:

1. **检测API变更**
   ```typescript
   // 检查是否有API相关代码变更
   const apiChanges = detectAPIChanges(task.files);
   ```

2. **更新OpenAPI规范**
   ```yaml
   # 如果有API变更，更新 openapi.yaml
   paths:
     /api/xxx:
       post:
         # ...
   ```

3. **验证契约一致性**
   ```bash
   # 运行契约测试
   npm run contract:test
   ```

4. **结果处理**
   - 契约验证通过 → 继续
   - 契约验证失败 → 标记任务失败，报告差异

## 执行命令

### 启动执行

```
# 手动执行（每个任务需确认）
task-executor --plan docs/v1/03-dev-plan/01-backend-mod-01.md

task-executor --plan docs/v1/03-dev-plan/01-backend-mod-01.md --auto

task-executor --all --full-auto
```

### 恢复执行

```
# 从断点恢复
task-executor --resume

# 查看当前状态
task-executor --status
```

### 查看进度

```
# 显示执行进度
task-executor --progress

# 显示详细任务列表
task-executor --tasks
```

## 执行约束

### 硬性约束（必须遵守）

| 约束 | 限制 | 超限处理 |
|-----|------|---------|
| 代码变更行数 | ≤ 200行 | 停止执行，报告超限 |
| 涉及文件数 | ≤ 5个 | 停止执行，报告超限 |
| 测试用例数 | ≤ 10个 | 停止执行，报告超限 |
| 执行时间 | ≤ 30分钟 | 暂停执行，询问用户 |

### 软性约束（建议遵守）

| 约束 | 建议 | 说明 |
|-----|------|------|
| 函数长度 | ≤ 50行 | 保持代码可读性 |
| 圈复杂度 | ≤ 10 | 保持逻辑简单 |
| 嵌套深度 | ≤ 4层 | 保持结构清晰 |
