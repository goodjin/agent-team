# Agent Team 需求文档（任务中心版）

**版本**: 1.0.0
**主题**: 以任务为中心的多智能体协作系统

---

## 1. 核心理念

### 1.1 任务即中心

系统围绕**任务**展开一切活动：

```
用户创建任务
    ↓
任务智能拆分
    ↓
多智能体协作执行
    ↓
执行过程记录日志
    ↓
产出成品交付
```

### 1.2 四个核心维度

| 维度 | 说明 | 体现形式 |
|------|------|----------|
| **执行过程** | 任务如何被执行 | 执行日志、步骤追踪 |
| **任务拆分** | 复杂任务如何分解 | 子任务树、依赖关系 |
| **日志体现** | 全程可追溯 | 结构化日志、时间线 |
| **成品体现** | 任务产出什么 | 代码、文档、报告等 |

## 2. 任务模型

### 2.1 任务定义

```typescript
interface Task {
  // 基础信息
  id: string;
  title: string;
  description: string;
  status: TaskStatus;

  // 执行配置
  role: string;           // 执行角色（如：前端开发、架构师）
  parentId?: string;      // 父任务ID（用于子任务）
  dependencies?: string[]; // 依赖的其他任务

  // 时间追踪
  createdAt: Date;
  startedAt?: Date;
  completedAt?: Date;

  // 执行结果
  artifacts: Artifact[];  // 产出的成品
  logs: LogEntry[];       // 执行日志
}

type TaskStatus =
  | 'pending'      // 待执行
  | 'running'      // 执行中
  | 'paused'       // 已暂停
  | 'completed'    // 已完成
  | 'failed';      // 失败
```

### 2.2 任务生命周期

```
[pending] → [running] → [completed]
               ↓
            [paused] → [running]
               ↓
            [failed] → [pending] (重试)
```

## 3. 执行过程

### 3.1 执行流程

```
1. 任务接收
   └─→ 解析任务描述，识别意图

2. 任务分析
   └─→ 评估复杂度，决定是否需要拆分

3. 任务拆分（可选）
   └─→ 拆分为多个子任务，建立依赖关系

4. 任务分配
   └─→ 根据角色匹配合适的智能体

5. 并行执行
   └─→ 无依赖的子任务同时执行

6. 结果汇总
   └─→ 收集各子任务结果，整合输出

7. 成品交付
   └─→ 保存代码、文档等产出物
```

### 3.2 执行状态追踪

每个任务执行时实时记录：

- **当前步骤**: 正在执行的具体操作
- **已用时间**: 从开始到当前的时间
- **Token消耗**: LLM调用成本
- **工具调用**: 使用了哪些工具
- **中间产物**: 执行过程中产生的临时文件

## 4. 任务拆分

### 4.1 拆分策略

| 策略 | 适用场景 | 示例 |
|------|----------|------|
| **按角色拆分** | 需要多角色协作 | 产品分析 → 架构设计 → 编码实现 |
| **按模块拆分** | 功能独立的模块 | 用户模块 + 订单模块 + 支付模块 |
| **按步骤拆分** | 有明确先后顺序 | 先设计数据库 → 再写API → 最后写前端 |

### 4.2 子任务依赖

```typescript
interface TaskDependency {
  taskId: string;           // 依赖的任务ID
  type: 'finish-to-start'   // 完成即开始（默认）
       | 'finish-to-finish' // 完成到完成
       | 'start-to-start';  // 开始到开始
}
```

### 4.3 拆分示例

**原始任务**: "开发一个电商网站的购物车功能"

```
购物车功能
├── [子任务1] 设计购物车数据模型
│     角色: 架构师
│     产出: 数据模型设计文档
│
├── [子任务2] 实现购物车API接口
│     角色: 后端开发
│     依赖: 子任务1
│     产出: API代码 + 单元测试
│
├── [子任务3] 实现购物车前端页面
│     角色: 前端开发
│     依赖: 子任务2
│     产出: 前端组件代码
│
└── [子任务4] 编写接口测试
      角色: 测试工程师
      依赖: 子任务2
      产出: 测试用例
```

## 5. 日志体现

### 5.1 日志类型

| 类型 | 记录内容 | 用途 |
|------|----------|------|
| **系统日志** | 任务状态变更、调度决策 | 问题排查 |
| **执行日志** | 智能体思考过程、工具调用 | 过程追溯 |
| **操作日志** | 用户操作、配置变更 | 审计追踪 |

### 5.2 日志结构

```typescript
interface LogEntry {
  timestamp: Date;
  level: 'debug' | 'info' | 'warn' | 'error';
  taskId: string;
  agentId?: string;

  // 日志内容
  type: 'thought'      // 智能体思考
      | 'action'       // 执行动作
      | 'tool_call'    // 工具调用
      | 'tool_result'  // 工具结果
      | 'milestone'    // 里程碑
      | 'error';       // 错误

  content: string;
  metadata?: Record<string, any>;
}
```

### 5.3 时间线视图

## 6. 成品体现

### 6.1 成品类型

```typescript
interface Artifact {
  id: string;
  taskId: string;
  type: 'code'        // 源代码文件
      | 'document'    // 文档（Markdown、Word等）
      | 'diagram'     // 图表（架构图、流程图）
      | 'test'        // 测试用例/报告
      | 'config'      // 配置文件
      | 'data';       // 数据文件（JSON、CSV等）

  name: string;
  path: string;       // 存储路径
  size: number;
  createdAt: Date;
}
```

### 6.2 成品管理

- **自动收集**: 智能体将产出文件自动关联到任务
- **版本追踪**: 同一文件的多次修改保留历史
- **快速访问**: 从任务页面直接查看所有产出
- **导出打包**: 支持将任务成品打包下载

### 6.3 成品示例

| 任务类型 | 典型成品 |
|----------|----------|
| 需求分析 | PRD文档、用户故事、流程图 |
| 架构设计 | 架构图、接口定义、数据模型 |
| 编码实现 | 源代码、单元测试、配置文件 |
| 代码审查 | 审查报告、问题清单、建议文档 |
| 测试验证 | 测试用例、测试报告、覆盖率报告 |

## 7. 角色系统

### 7.1 内置角色

| 角色 | 职责 | 典型任务 |
|------|------|----------|
| **任务分析师** | 分析任务、决定拆分策略 | 所有任务的入口 |
| **产品经理** | 需求分析、PRD编写 | 需求相关任务 |
| **架构师** | 技术方案、架构设计 | 设计相关任务 |
| **后端开发** | API开发、业务逻辑 | 后端编码任务 |
| **前端开发** | UI实现、组件开发 | 前端编码任务 |
| **测试工程师** | 测试计划、用例执行 | 测试相关任务 |

### 7.2 角色匹配

```typescript
// 根据任务内容自动匹配角色
function matchRole(task: Task): string {
  if (task.description.includes('需求') || task.description.includes('PRD')) {
    return 'product-manager';
  }
  if (task.description.includes('架构') || task.description.includes('设计')) {
    return 'architect';
  }
  if (task.description.includes('前端') || task.description.includes('UI')) {
    return 'frontend-dev';
  }
  // ... 更多匹配规则
  return 'task-analyzer'; // 默认角色
}
```

## 9. API 接口

### 9.1 任务管理

```typescript
// 创建任务
POST /api/tasks
{
  "title": "开发登录功能",
  "description": "实现用户登录的完整功能",
  "role": "backend-dev"
}

// 获取任务详情（包含执行日志、子任务、成品）
GET /api/tasks/:id

// 获取任务执行日志
GET /api/tasks/:id/logs

// 获取任务成品
GET /api/tasks/:id/artifacts

// 列出子任务
GET /api/tasks/:id/subtasks

// 重试失败任务
POST /api/tasks/:id/retry
```

### 9.2 实时推送

```typescript
// WebSocket 实时推送任务状态变更
ws://localhost:3000/ws/tasks/:id

// 推送事件
{
  "type": "status_change",
  "timestamp": "2026-02-20T10:05:00Z",
  "data": {
    "oldStatus": "pending",
    "newStatus": "running"
  }
}
```

## 10. 数据存储

### 10.1 存储结构

```
data/
├── tasks/
│   ├── task-{id}.json      # 任务元数据
│   └── index.json          # 任务索引
├── logs/
│   └── task-{id}/
│       └── {timestamp}.log # 执行日志
├── artifacts/
│   └── task-{id}/
│       ├── code/           # 代码文件
│       ├── docs/           # 文档
│       └── tests/          # 测试文件
└── cache/
    └── agent-context/      # 智能体上下文缓存
```

### 10.2 持久化策略

- **任务数据**: 立即持久化，确保不丢失
- **日志数据**: 批量写入，每5秒或每100条刷新
- **成品文件**: 写入时立即持久化

## 12. 验收标准

### 12.1 功能验收

| 功能 | 验收标准 |
|------|----------|
| 任务创建 | 5秒内完成任务创建并返回ID |
| 任务执行 | 简单任务（无需拆分）60秒内完成 |
| 任务拆分 | 复杂任务自动拆分为2-5个子任务 |
| 并行执行 | 无依赖子任务同时执行，节省50%以上时间 |
| 日志完整 | 每个任务执行步骤都有日志记录 |
| 成品交付 | 代码/文档等产出自动关联到任务 |

### 12.2 性能指标

| 指标 | 目标值 |
|------|--------|
| 任务创建延迟 | < 100ms |
| 日志查询速度 | < 500ms（1000条日志） |
| 任务列表加载 | < 300ms |
| 并发任务数 | >= 10个 |
